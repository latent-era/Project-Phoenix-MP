<objective>
Improve cloud sync discoverability by moving the Cloud Sync card to the top of SettingsTab, and add a sync error indicator that shows when sync has failed 3+ consecutive times.
</objective>

<context>
- Phase context: `.planning/phases/31-polish-validation/31-CONTEXT.md`
- Branch: `MVP`
- Cloud Sync card is currently buried after 7+ sections in SettingsTab (near bottom)
- SyncTriggerManager exposes `hasPersistentError: StateFlow<Boolean>` (true after 3+ failures)
- Auto-sync triggers: ON_RESUME (5min throttle) + workout complete (immediate)
- Users have zero visibility into sync status from the main screen
- The Cloud Sync card was uncommented in Phase 29
</context>

<tasks>
### Task 1: Move Cloud Sync Card to Top of SettingsTab

**File**: `SettingsTab.kt`
**Action**: Cut the entire Cloud Sync Card section (the `Card` block starting with `// Cloud Sync Section - Material 3 Expressive`) and paste it at the TOP of the settings content — before the Donation Card or whatever the current first card is.

This makes Cloud Sync the first thing users see when they open Settings, dramatically improving discoverability.

**Verify**: Cloud Sync card appears at the top of the Settings scroll.

### Task 2: Add Sync Error Indicator

**File**: `SettingsTab.kt`
**Action**: Inside the Cloud Sync Card, after the description text, add a conditional error warning:

1. Inject SyncTriggerManager and observe hasPersistentError:
   ```kotlin
   val syncTriggerManager: SyncTriggerManager = koinInject()
   val hasSyncError by syncTriggerManager.hasPersistentError.collectAsState()
   ```

2. After the description Text, add:
   ```kotlin
   if (hasSyncError) {
       Spacer(modifier = Modifier.height(Spacing.small))
       Row(
           modifier = Modifier.fillMaxWidth(),
           verticalAlignment = Alignment.CenterVertically
       ) {
           Icon(
               Icons.Default.Warning,
               contentDescription = "Sync Error",
               tint = MaterialTheme.colorScheme.error,
               modifier = Modifier.size(16.dp)
           )
           Spacer(modifier = Modifier.width(Spacing.small))
           Text(
               "Sync error \u2014 tap above to retry",
               style = MaterialTheme.typography.bodySmall,
               color = MaterialTheme.colorScheme.error
           )
       }
   }
   ```

**Required imports**:
- `import androidx.compose.runtime.collectAsState`
- `import androidx.compose.material.icons.filled.Warning`
- `import com.devil.phoenixproject.data.sync.SyncTriggerManager`

**Verify**: Error indicator only shows when hasPersistentError is true.
